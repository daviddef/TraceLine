# TraceLine

An iOS game about drawing one continuous line. Two rules govern everything:

1. **Never lift your finger** — the instant contact breaks, the round ends.
2. **Lines can never cross** — the path cannot intersect itself.

Fill enough of the board before the clock runs out. From level 6, obstacles fall from
the top of the screen and the line has to weave around them.

Swift 5.9 / SpriteKit / iOS 16+. iPhone portrait. No image assets — every screen, the
line, the obstacles and even the app icon are drawn in code.

**Status:** v1 (World 1, levels 1–10) is on TestFlight as "TraceLine!".

## Quick start

The project is generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen), so
`TraceLine.xcodeproj` is build output and is not committed:

```sh
brew install xcodegen
cd TraceLine
xcodegen generate
open TraceLine.xcodeproj
```

Run the tests (39 unit + 4 UI) with:

```sh
cd TraceLine
xcodebuild -project TraceLine.xcodeproj -scheme TraceLine \
  -sdk iphonesimulator -destination 'platform=iOS Simulator,name=iPhone 17' test
```

## How it hangs together

`DrawingEngine` owns both rules and has no SpriteKit dependency: `GameScene` feeds it
touch points and obstacle descriptors, and it answers with a `DrawResult`. That
separation is what makes the rules unit-testable without a running scene — see
`TraceLineTests/DrawingEngineTests.swift`.

Self-crossing is detected with a standard segment-intersection test against every prior
segment, skipping the immediately preceding one (it shares an endpoint and would always
report a hit). Obstacle collision tests the whole proposed segment rather than just its
endpoint, so a fast flick cannot tunnel through an obstacle between two touch events.
